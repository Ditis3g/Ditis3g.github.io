오늘은 직접 코드를 거의 짜지 않고도 무료로 내 블로그를 만들 수 있다는 걸 체험했다. GitHub에 가입하는 것부터 시작해 실제로 사이트가 인터넷에 뜨기까지의 과정을 정리해 둔다.

## GitHub Pages로 무료 사이트 띄우기

먼저 GitHub에 가입하고 저장소를 하나 만들었다. 이때 가장 중요한 규칙은 **저장소 이름을 정확히 `사용자이름.github.io` 형식으로 지어야 한다**는 것이다. 이 규칙을 지켜야만 무료 주소가 자동으로 활성화된다.

GitHub Pages는 내 저장소에 올린 파일을 무료 웹사이트로 띄워 주는 서비스다. 별도의 서버를 빌릴 필요가 없으니 서버비가 0원이라는 점이 핵심이다.

설정은 저장소의 **Settings > Pages** 메뉴에서 발행을 켜면 된다. 이때 Source를 **GitHub Actions**로 바꿔 주는 것이 포인트다. 그래야 뒤에서 설명할 Hugo의 자동 배포가 동작한다.

## Hugo와 PaperMod 테마

Hugo는 마크다운으로 글을 쓰면 자동으로 블로그 페이지를 만들어 주는 **정적 사이트 생성기**다. 글만 마크다운으로 작성하면 나머지 HTML 페이지 생성은 Hugo가 알아서 처리해 준다.

디자인은 직접 만들지 않고 **PaperMod** 테마를 가져다 썼다. 테마는 git submodule 형태로 저장소에 추가했다. 미리 잘 만들어진 테마를 붙이는 것만으로 깔끔한 블로그 외형을 갖출 수 있었다.

## Git으로 올리기, 그리고 403 에러

작업한 내용을 깃허브에 올리는 흐름도 정리하면 간단하다.

- `git commit` = 변경 사항 저장
- `git push` = 깃허브로 올리기

다만 push 과정에서 한 번 막혔다. 컴퓨터에 **다른 계정의 자격 증명(credential)이 저장**되어 있으면 권한이 없어 **403 에러**가 난다. 저장된 자격 증명을 정리해 내 계정으로 맞춰 주니 문제가 해결됐다.

## 한 줄 정리

코드를 거의 짜지 않고도 GitHub Pages + Hugo + PaperMod 조합으로 0원에 진짜 블로그를 띄울 수 있었다.
