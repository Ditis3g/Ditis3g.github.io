SO-ARM101과 LeRobot로 로봇 학습을 시작하면 가장 먼저 마주치는 단어가 **정책(Policy)** 입니다. ACT, Diffusion Policy, SmolVLA — 이름은 제각각이지만 결국 하는 일은 같습니다. 이 글에서는 정책이라는 개념부터 세 모델의 차이, 그리고 이들을 가르는 비전 정책과 VLA의 구분, 마지막으로 로봇 행동을 뽑아내는 두 방식까지 한 번에 정리합니다.

## 정책(Policy): 관찰을 행동으로 바꾸는 함수

로봇 학습에서 말하는 정책은 현재 관찰을 보고 다음 행동을 결정하는 함수입니다. 더 명확히 말하면 **어떤 입력을 받아 어떤 출력을 낼지 결정하는 함수**입니다.

```text
행동 = π(관찰)
```

| 개념 | 의미 |
|---|---|
| 입력 | 관찰, 즉 카메라 영상 + 관절 상태 |
| 출력 | 행동, 즉 관절 목표각이나 그리퍼 명령 |
| 목적 | 현재 상황에서 로봇이 무엇을 해야 할지 결정 |

실제 로봇에서는 이 함수가 아주 빠른 주기로 반복됩니다.

```text
카메라 영상 + 관절 상태
→ 정책 모델
→ 다음 관절 목표값 출력
→ 모터 이동
→ 초당 수십 번 반복
```

여기서 핵심은, ACT · Diffusion Policy · SmolVLA가 모두 결국 `관찰 → 행동`을 만드는 정책이라는 점입니다. 세 모델의 차이는 이 정책 함수를 **어떤 방식으로 만들었느냐**에 있습니다.

## 세 가지 정책 비교: ACT · Diffusion Policy · SmolVLA

먼저 전체 그림을 한 표로 봅니다.

| 항목 | ACT | Diffusion Policy | SmolVLA |
|---|---|---|---|
| 정식 명칭 | Action Chunking with Transformers | Diffusion Policy | Small Vision-Language-Action |
| 분류 | 비전 정책 | 비전 정책 | VLA |
| 구조 | Transformer + CVAE | 확산 모델 | VLM 백본 + Action Expert |
| 행동 생성 | 미래 행동 청크 예측 | 노이즈를 디노이징 | Flow matching |
| 언어 명령 | ❌ | ❌ | ✅ |
| 사전학습 | ❌ | ❌ | ✅ |
| 크기 | 작음 | 중간 | 약 450M |
| 속도 | 매우 빠름 | 느릴 수 있음 | 빠른 편 |
| 일반화 | 낮음 | 낮음 | 높음 |
| SO-ARM101 입문성 | ✅ 좋음 | △ 가능 | △ 더 복잡 |
| VLA 여부 | ❌ | ❌ | ✅ |

성격만 뽑으면 이렇습니다.

| 모델 | 성격 |
|---|---|
| ACT | 가볍고 빠른 기본 베이스라인 |
| Diffusion Policy | 여러 정답이 가능한 상황에 강함 |
| SmolVLA | 언어 명령과 일반화에 강한 경량 VLA |

### ACT — 가볍고 빠른 비전 정책

ACT는 가장 단순하게 말하면 **현재 장면을 보고, 앞으로 몇 순간 동안의 행동 묶음을 한 번에 예측하는 모델**입니다. 예를 들어 수건 접는 시연을 50번 보여주면 다음처럼 동작합니다.

```text
카메라 장면 + 로봇 팔 상태
→ 다음 1초 동안의 관절 움직임 예측
```

| 항목 | 내용 |
|---|---|
| 입력 | 카메라 영상 + 관절 상태 |
| 출력 | 행동 청크 |
| 언어 명령 | ❌ 없음 |
| 학습 방식 | 시연 데이터로 모방 학습 |
| 장점 | 빠름, 가벼움, 실습에 좋음 |
| 단점 | 태스크 일반화 약함 |

ACT는 로봇을 실제로 움직이므로 **피지컬 AI**는 맞지만, 언어 명령과 VLM 백본이 없으므로 **VLA는 아닙니다.**

```text
ACT = 피지컬 AI ✅ / VLA ❌
```

### Diffusion Policy — 여러 정답에 강한 비전 정책

Diffusion Policy는 행동을 바로 찍어내지 않습니다. 대신 랜덤한 행동 숫자에서 시작해 여러 번 고쳐 그럴듯한 행동을 완성합니다.

```text
랜덤한 행동 숫자
→ 여러 번 고침
→ 그럴듯한 행동 완성
```

이 방식이 강점을 발휘하는 건 정답이 하나가 아닐 때입니다. 컵을 잡는 상황을 생각해 보면,

```text
왼쪽으로 돌아가도 됨
오른쪽으로 돌아가도 됨
위에서 잡아도 됨
옆에서 잡아도 됨
```

정답 행동이 여러 개일 수 있습니다. ACT는 이런 경우 여러 행동을 평균 내다가 애매한 행동을 만들 수 있지만, Diffusion Policy는 가능한 행동 중 하나를 더 자연스럽게 뽑아냅니다.

| 항목 | 내용 |
|---|---|
| 입력 | 카메라 영상 + 관절 상태 |
| 출력 | 디노이징된 행동 |
| 언어 명령 | ❌ 없음 |
| 장점 | 멀티모달 상황에 강함 |
| 단점 | 여러 번 고쳐야 해서 느릴 수 있음 |

```text
Diffusion Policy = 피지컬 AI ✅ / VLA ❌
```

### SmolVLA — 언어 명령을 받는 경량 VLA

SmolVLA는 앞의 두 모델과 다르게 **VLA**입니다. 이미지 보기 + 언어 명령 이해 + 로봇 행동 출력을 모두 연결한 모델입니다.

```text
카메라 영상 + "pick up the cup"
→ VLM이 장면과 명령 이해
→ Action Expert가 실제 행동 숫자 생성
→ 로봇 실행
```

여기서 흔한 오해 하나를 짚고 넘어갈 필요가 있습니다. SmolVLA가 말을 이해한다고 해서, 배우지 않은 동작을 갑자기 다 할 수 있는 것은 아닙니다.

```text
"fold the towel" → 수건 접기 데이터를 배웠으면 가능
"pick up the cup" → 컵 집기 데이터를 안 배웠으면 어려움
```

언어 명령은 마법이 아니라 **여러 동작 중 무엇을 할지 고르는 라벨(스위치)** 에 가깝습니다. 그래서 언어와 행동을 함께 학습해야 명령이 제대로 스위치처럼 작동합니다.

```text
"fold the towel"      → 수건 접기
"pick up the cup"     → 컵 집기
"put block in box"    → 블록 넣기
```

```text
SmolVLA = 피지컬 AI ✅ / VLA ✅
```

## VLA는 무엇이 다른가: 피지컬 AI · 비전 정책 · VLA · VLM

세 모델을 하나의 지도로 정리하면 이렇게 됩니다.

```text
피지컬 AI
│
├─ 비전 정책
│  ├─ ACT              → 피지컬 AI ✅ / VLA ❌
│  └─ Diffusion Policy → 피지컬 AI ✅ / VLA ❌
│
└─ VLA
   └─ SmolVLA          → 피지컬 AI ✅ / VLA ✅
```

| 개념 | 뜻 | 예시 |
|---|---|---|
| 피지컬 AI | 물리 세계에서 인식하고 행동하는 AI 전체 | ACT, Diffusion, SmolVLA |
| 비전 정책 | 언어 없이 관찰만 보고 행동하는 정책 | ACT, Diffusion |
| VLA | 비전 + 언어 + 행동을 연결한 모델 | RT-2, OpenVLA, π0, SmolVLA |

### VLA가 특별한 이유는 VLM 백본이다

VLA가 특별한 이유는 단순히 언어 명령을 받기 때문만은 아닙니다. VLA를 글자 그대로 풀면 `Vision + Language + Action`, 즉 이미지 보기 + 언어 명령 이해 + 로봇 행동 출력을 연결한 모델입니다. 그런데 요즘 연구에서 말하는 **현대적 의미의 VLA는 여기서 한 단계 더 좁게** 봅니다. 핵심은 **사전학습된 VLM 백본을 쓴다**는 점입니다.

```text
넓은 의미의 VLA
= Vision + Language + Action을 연결한 모델

좁은 의미, 현대적 의미의 VLA
= Vision + Language + Action + 사전학습된 VLM 백본
```

여기서 VLM은 **Vision-Language Model**, 즉 이미지를 보고 언어로 이해할 수 있는 모델입니다.

```text
이건 컵이다
이건 빨간 블록이다
수건은 접을 수 있다
상자는 물건을 넣는 곳이다
```

이런 지식은 보통 인터넷 이미지와 텍스트 데이터에서 미리 배운 것입니다. VLM 백본은 이 웹에서 배운 **시각·언어 상식**을 로봇에 가져오기 때문에, 로봇 데이터에 적게 나온 물체나 명령도 더 잘 이해할 수 있습니다.

```text
웹에서 배운 세상 상식
→ 장면과 명령 이해
→ 로봇 행동 생성에 활용
```

정리하면 VLA의 구조는 이렇게 나뉩니다.

```text
VLM 백본     → 장면과 명령 이해
Action Expert → 실제 로봇 행동 숫자 생성

VLA = VLM 백본 + 행동 출력기
```

### RT-1은 어디에 속하나 — proto-VLA

이 구분이 필요해진 대표적인 계기가 **RT-1**입니다. RT-1은 언어 명령을 받습니다.

```text
카메라 영상 + "사과를 집어"
→ RT-1
→ 행동 출력
```

그래서 ACT나 Diffusion Policy처럼 언어가 없는 비전 정책은 아닙니다. 하지만 RT-2처럼 강한 VLM 백본을 로봇 행동에 연결한 현대적 VLA도 아닙니다. 그래서 RT-1은 **VLA 직전 모델, proto-VLA** 정도로 봅니다.

| 모델 | 언어 입력 | VLM 백본 | 분류 |
|---|---|---|---|
| RT-1 | ✅ 있음 | ❌ 없음, 로봇 데이터 중심 | VLA 직전, proto-VLA |
| RT-2 | ✅ 있음 | ✅ 있음 | 최초의 현대적 VLA |

### 모델 계보와 타임라인

| 시기 | 모델 | 분류 |
|---|---|---|
| 2022.12 | RT-1 | 언어 조건화 모델, proto-VLA |
| 2023.03 | Diffusion Policy | 비전 정책 |
| 2023.04 | ACT | 비전 정책 |
| 2023.07 | RT-2 | 최초의 현대적 VLA |
| 2024 | OpenVLA, π0 | VLA |
| 2025 | SmolVLA | 경량 VLA |

한 가지 오해를 바로잡을 필요가 있습니다. `ACT → Diffusion → VLA`처럼 순차 계승 관계가 **아닙니다.** ACT와 Diffusion Policy는 거의 같은 시기의 비전 정책이고, Diffusion Policy가 살짝 먼저입니다. 또 SmolVLA는 첫 VLA가 아니며, 현대적 의미의 VLA는 보통 **RT-2부터**로 봅니다.

## 행동을 만드는 두 방식: Action Token vs Action Expert

VLM에서 실제 로봇 행동을 뽑아내는 방식은 크게 두 가지입니다.

| 항목 | Action Token | Action Expert (Flow Matching) |
|---|---|---|
| 대표 모델 | RT-2, base OpenVLA | π0, SmolVLA |
| 행동 표현 | discrete token | continuous action |
| 누가 행동을 만드나 | VLM/LLM이 토큰 출력 | 별도 action expert가 숫자 생성 |
| 변환 과정 | token → de-tokenize → action | 바로 action |
| 정밀도 | 칸 단위라 손실 있음 | 연속값이라 정밀 |
| 속도 | 토큰 순차 출력이면 느림 | chunk/병렬 생성 가능 |
| 구조 | 단순 | 복잡 |
| 제어 품질 | 거칠 수 있음 | 부드럽고 빠른 제어에 유리 |

### Action Token — 번호표를 고르는 방식

Action Token 방식은 행동 숫자를 직접 만드는 것이 아니라, 미리 나눠둔 번호표 중 하나를 고르는 방식입니다.

```text
이미지 + 언어
→ VLM이 action token 출력
→ 디토크나이저가 token을 실제 action 숫자로 변환
→ 모터/컨트롤러 실행
```

```text
컵이 왼쪽 10cm에 있음
→ 가장 맞는 행동 토큰 선택
→ 130번 토큰
→ 130번을 실제 숫자로 변환
```

즉 `번호표 선택 → 숫자로 변환`입니다.

### Action Expert (Flow Matching) — 숫자를 직접 만드는 방식

Action Expert는 행동 숫자를 직접 만듭니다. SmolVLA의 Action Expert는 이때 **Flow Matching** 방식으로 행동을 생성합니다.

```text
카메라 + "수건을 접어"
→ VLM이 상황 이해
→ Action Expert가 연속 action 숫자 생성
→ 모터/컨트롤러 실행
```

핵심은 랜덤한 행동 숫자에서 시작해 정답 방향으로 조금씩 수정해 최종 행동을 완성한다는 점입니다. 네트워크가 매번 **어느 방향으로 고쳐야 하는지**를 알려줍니다.

```text
시작: 랜덤 81
→ 56
→ 38
→ 31
→ 30 ✅
```

컵이 왼쪽 10cm에 있다면 이렇게 수렴합니다.

```text
처음 후보: 왼쪽 3cm
→ 수정: 왼쪽 7cm
→ 수정: 왼쪽 9.5cm
→ 최종: 왼쪽 10.27cm
```

두 방식을 한 줄로 대비하면 이렇습니다.

```text
Action Token  = 언어 모델에 행동까지 토큰으로 맡김
Action Expert = 언어 이해와 행동 생성을 분업
```

## 실습 관점에서의 선택과 핵심

SO-ARM101이나 LeRobot 실습 관점에서 고르는 기준은 단순합니다.

```text
처음 실습하기 좋음            → ACT
여러 가능한 행동을 잘 다루고 싶음 → Diffusion Policy
언어 명령과 일반화를 다루고 싶음  → SmolVLA
```

전체를 관통하는 세 가지만 기억하면 됩니다.

첫째, **정책 = 관찰 → 행동 함수**입니다. 세 모델의 차이는 정책을 만드는 방식뿐입니다. 둘째, **VLA는 계산보다 모방에 가깝습니다.** 똑똑해 보여도 핵심은 여전히 데이터이고, 배운 범위 밖은 약합니다. 셋째, **언어 입력 통로가 확장성을 만듭니다.** 언어 명령은 여러 동작을 고르는 스위치라서, 여러 작업을 하나의 모델에 묶기 쉽게 해 줍니다.

## 한 줄 정리

세 모델은 모두 `관찰 → 행동`을 만드는 정책이며, 차이는 만드는 방식뿐이다 — ACT·Diffusion Policy는 언어 없는 비전 정책이고, SmolVLA는 사전학습된 VLM 백본에 Action Expert를 붙여 언어 명령까지 소화하는 경량 VLA다.
